## 🇬🇧 English

### Documentation policy

All GitHub documentation is maintained in **English + Russian**:

- `README.md` (🇬🇧) and `README.ru.md` (🇷🇺) are kept in sync.
- Each major doc may have a `.ru.md` counterpart when it benefits contributors.
- The top of every bilingual doc links to its other-language version.
- Code comments, commit messages, and identifiers stay in **English** (universal for contributors).

### Bot & dashboard localization (i18n-first)

The bot UI targets **6 launch languages**:

| Language | Code | Discord locale | Priority |
|----------|------|----------------|:--------:|
| 🇬🇧 English | `en` | `en-US`, `en-GB` | Tier 1 |
| 🇷🇺 Русский | `ru` | `ru` | Tier 1 |
| 🇺🇦 Українська | `uk` | `uk` | Tier 2 |
| 🇪🇸 Español | `es` | `es-ES`, `es-419` | Tier 2 |
| 🇩🇪 Deutsch | `de` | `de` | Tier 2 |
| 🇫🇷 Français | `fr` | `fr` | Tier 2 |

### Architecture

**Language resolution order** (first match wins):
1. Per-user preference (dashboard / profile setting) — *future*
2. Per-guild setting (`GuildSettings.locale`)
3. Discord interaction `locale` (auto-detected from the user's client)
4. Fallback: `en`

**Where translations live:**
- A dedicated `@sailorclawbot/i18n` package (planned) exporting a typed `t(key, locale, vars?)` function.
- Translation catalogs as JSON/TS per locale: `packages/i18n/locales/<code>.json`.
- Keys are namespaced by domain: `moderation.warn.success`, `economy.daily.claimed`, etc.
- Type-safe keys — a missing key is a compile error, never a runtime surprise.

**Principles:**
- **i18n-first**: no user-facing string is hardcoded in services or the bot layer. Domain services return *codes/data*; the bot/dashboard layer renders localized text.
- ICU MessageFormat for plurals & interpolation (e.g. "{count, plural, one {# warning} other {# warnings}}").
- Discord slash-command names/descriptions use Discord's built-in `name_localizations` / `description_localizations`.
- Dates, numbers, currency via `Intl` with the resolved locale.

## 🇷🇺 Русский

### Политика документации

Вся документация на GitHub ведётся на **английском + русском**:

- `README.md` (🇬🇧) и `README.ru.md` (🇷🇺) поддерживаются синхронно.
- У крупных документов может быть версия `.ru.md`, когда это полезно участникам.
- Вверху каждого двуязычного документа — ссылка на версию на другом языке.
- Комментарии в коде, сообщения коммитов и идентификаторы остаются на **английском** (универсально для контрибьюторов).

### Локализация бота и дашборда (i18n-first)

Интерфейс бота рассчитан на **6 языков к запуску**:

| Язык | Код | Локаль Discord | Приоритет |
|------|-----|----------------|:---------:|
| 🇬🇧 English | `en` | `en-US`, `en-GB` | Tier 1 |
| 🇷🇺 Русский | `ru` | `ru` | Tier 1 |
| 🇺🇦 Українська | `uk` | `uk` | Tier 2 |
| 🇪🇸 Español | `es` | `es-ES`, `es-419` | Tier 2 |
| 🇩🇪 Deutsch | `de` | `de` | Tier 2 |
| 🇫🇷 Français | `fr` | `fr` | Tier 2 |

### Архитектура

**Порядок определения языка** (срабатывает первое совпадение):
1. Предпочтение пользователя (настройка в дашборде / профиле) — *в будущем*
2. Настройка сервера (`GuildSettings.locale`)
3. Локаль взаимодействия Discord (`locale`, автоопределение из клиента пользователя)
4. Запасной вариант: `en`

**Где живут переводы:**
- Отдельный пакет `@sailorclawbot/i18n` (запланирован) с типизированной функцией `t(key, locale, vars?)`.
- Каталоги переводов в JSON/TS по локалям: `packages/i18n/locales/<код>.json`.
- Ключи разбиты по доменам: `moderation.warn.success`, `economy.daily.claimed` и т.д.
- Типобезопасные ключи — отсутствующий ключ это ошибка компиляции, а не сюрприз в рантайме.

**Принципы:**
- **i18n-first**: ни одна строка для пользователя не зашита в сервисах или слое бота. Доменные сервисы возвращают *коды/данные*; слой бота/дашборда рендерит локализованный текст.
- ICU MessageFormat для множественного числа и подстановок (например «{count, plural, one {# предупреждение} few {# предупреждения} other {# предупреждений}}»).
- Имена/описания slash-команд используют встроенные `name_localizations` / `description_localizations` Discord.
- Даты, числа, валюта — через `Intl` с выбранной локалью.
